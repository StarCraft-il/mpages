A calm scrolling text sample

This demo is built for a mobile screen first. It keeps the important reading area stable, uses soft fading at the top and bottom, and moves the text with a GPU-friendly transform instead of forcing the browser to scroll the entire page.

Automatic scrolling can feel uncomfortable when it is too fast, when the whole screen moves, or when the movement jumps between frames. This version avoids that by moving only the text layer inside a fixed reader window. The rest of the interface remains still, so the eyes have a stable frame of reference.

Why the reader window helps

A full page scroll makes everything move: the header, the content, the background, and often the browser interface. That can create a feeling of flicker, especially on smaller screens. A contained reader is gentler because only the text moves while the surrounding card remains fixed.

The fade at the top and bottom hides the harsh moment where lines enter or leave the visible area. The focus band gives the eye a comfortable middle zone. You can change the focus size using the controls panel.

For comfortable reading, use a slow speed, a generous line height, and a reader window that does not occupy the entire screen.

Mobile-first behavior

On a phone, the layout uses one column, large touch targets, and a compact controls panel. The panel can be hidden when you want the reading area to feel clean. When hidden, a small button remains in the corner so the controls are easy to bring back.

On wider screens, the same controls move to a compact floating panel. The reader itself keeps a comfortable maximum width so the line length does not become too wide. This is useful because long lines make it harder to track the next line while reading.

Speed and direction

The scroll speed is measured in pixels per second. Low values feel more like a teleprompter. Higher values are useful when testing or scanning. The direction button lets the content move down or back up.

When the text reaches the end, it pauses briefly and then reverses direction. This avoids the visual shock of instantly jumping from the bottom back to the top. You can still use the reset button whenever you want to return to the beginning.

Text size and line height

The text size control changes the reading text without changing the interface. Larger text can be easier to read, but it also means fewer words fit in the window. The line height control changes the spacing between lines. A little extra spacing usually makes moving text more comfortable.

If the motion feels tiring, first lower the speed. Then increase line height slightly. After that, adjust the focus band until the middle of the reading window feels calm and stable.

Reader window height

The window height control changes how much of the screen is used for the scrolling text. A smaller window gives the eyes a tighter target. A taller window shows more context. On mobile, a middle value often feels best because it leaves enough room for the interface while keeping the text area predictable.

The goal is not only smooth animation. The goal is stable visual composition. When the background, borders, and controls remain still, the moving text is easier to follow.

Manual control

The position slider lets you move through the content manually. Using it automatically pauses the reader so you can inspect a specific paragraph. Press play again to continue automatic movement from the current position.

You can also drag directly inside the reading window on touch screens. This gives you a simple way to correct the position without relying only on the slider.

More sample text

Smooth scrolling is especially useful for speeches, scripts, presentations, reading practice, guided meditation text, lyrics practice, study notes, long instructions, or any situation where the reader wants a steady pace without constantly touching the screen.

The best settings depend on the content. Dense technical material should usually move slowly. Familiar text can move faster. Large text needs more vertical space. Short lines need less effort to track.

For a teleprompter-like setup, hide the controls, increase the text size, set the reader height high, and use a slow scroll speed. For a reading practice setup, keep the controls visible and use the position slider to repeat difficult sections.

The implementation uses `requestAnimationFrame`, which lets the browser synchronize the animation with the screen refresh rate. The content is moved with `translate3d`, which is usually smoother than repeatedly changing layout properties.

The text includes extra padding before the first paragraph and after the last paragraph. That lets the beginning and ending pass through the focus band naturally instead of starting or stopping abruptly at the edge of the visible area.

Final section

This final section gives you enough content to see the scrolling behavior clearly. Try reducing the focus band size, increasing the line height, and setting the speed to a very low value. The result should feel calm, predictable, and easy to follow.

When you are satisfied with the settings, hide the controls panel. The scrolling text will continue inside the reader window, and the screen will remain visually stable.
